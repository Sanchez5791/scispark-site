这是演示 · Demo only. 下面的题目、分数、等级都是样例,用来给老板看「学生分等级」整个流程长什么样。 真实账号不会用这些假数据 — 没做过定级 / 没答过题的学生,家长台会老实显示「暂无」。真题与三档提示由内容房/工厂房供。
Student Level System · Plan B学生分等级 · 方案 B

One walk-through, four steps 一条龙走完:入学定级 → 课中按等级给难度+提示 → 跟进 → 家长台

Plan B = difficulty + hints by level. Gamification (Plan C) comes later. 方案 B = 按等级给「题目难度 + 提示量」。游戏化(方案 C)以后再加,这次只预留接口。

Level · 当前等级
Sample question · 样题(states of matter)
Water freezes into ice. Is this a physical or chemical change? 水结成冰。这是物理变化还是化学变化?
Explain why melting ice is a physical change, using the idea of particles. 用「粒子」的概念解释:为什么冰融化是物理变化?
A student claims "melting" and "burning" are both just changes of state. Evaluate this claim and correct any error. 有同学说「融化」和「燃烧」都只是状态变化。评价这个说法并指出错误。
Hint · 提示(多) 一步步想:① 物理变化里有没有产生新物质?② 冰和水是不是同一种物质(H₂O)?③ 能不能变回去?能变回去、没新物质 = 物理变化。
Hint · 提示(中) 想一个关键问题:这个过程有没有生成新物质?粒子本身变了,还是只是排列/距离变了?
Hint · 提示(少) 一个词:可逆性。
03

Follow-up学习中跟进(C 混合)

Every answer is scored in the background (rolling). At the end of each unit the machine re-checks the level. The backend level can go up or down; the shown level only goes up. 平时每答一题后台滚动记分;每学完一个单元,机器正式重算一次等级。后台真实等级可升可降(用来调难度);给学生/家长看的显示等级只升不降。

Backend · 后台真实等级
L2
机器用它决定给什么难度的题 + 提示量。
可升可降。例:这次单元考差了 → 默默降到 L1,给简单点的题。
Shown · 显示等级(学生/家长看)
L2
= 历史最高,只升不降。后台掉到 L1,这里仍显示 L2,绝不显示「降级」。
Students and parents never see a downgrade — only their highest level so far. 学生和家长永远看不到「降级」,只看到自己到过的最高等级。(死规则)
