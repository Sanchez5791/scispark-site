2. WHY THIS FILE EXISTS (FAILURE PREVENTION)
================================================================

This file was added 2026-04-27 after a working AI mis-built Year 7
with 21 questions instead of 27. Root causes of that failure:

R1  Y7 had Y7_ENTRY_FINAL_BLUEPRINT_LOCK.txt (locks marks) but had
    NO STRUCTURE LOCK file (locks question count). The AI inferred
    question count from marks alone and chose a wrong granularity
    (5 multi-mark items in Part B; should have been 15 single-mark
    items).

R2  Y8 already had a STRUCTURE LOCK file with the right pattern
    (5+15+3+4=27). The AI did not look at Y8 to mirror its pattern.

R3  The AI ran Phase 3 (Source Mapping) without first producing a
    one-line "architecture confirmation" with the user. There was
    no checkpoint that would force the question-count error to
    surface before downstream work began.

R4  Source mapping v1 file did not have a top-line architecture
    summary. AIs reading downstream files could not detect the
    error.

R5  The AI did not list its assumptions ("21 questions because each
    mark is one question is my assumption") before committing.

    Phase 3 (Source Mapping) cannot start until this file exists
    AND user has signed off "structure approved" in writing.

P3  MIRROR-BEFORE-INVENT RULE
    Before designing structure for any Year, the AI must check if
    a sibling Year (Y6/Y8/Y9) already has a STRUCTURE LOCK and
    mirror it as the default. Inventing a new structure requires
    explicit user instruction.

P4  ARCHITECTURE-CHECKPOINT BEFORE WORK
    Phase 1 must produce a 3-line "Architecture Confirmation":
      Line 1: total questions = N
      Line 2: per-part counts = a + b + c + d
      Line 3: per-part marks  = A + B + C + D
    The user signs off these 3 lines explicitly. No work proceeds
    without this.

P5  ASSUMPTION-LISTING DUTY
    When choosing question granularity (e.g. one 5m big question
    vs five 1m small questions), the AI must explicitly list the
    assumption and ask before committing. Treat every "I picked X
    because that seemed right" as a flag to stop and confirm.
